Système de gestion de base de données (SGBD)

Definitions

Une base de données peut se définir comme un ensemble organisé de données devant etre manipulées à l'aide d'un ensemble de programme.

Un système de gestion de base de données (SGBD) est le logiciel qui permet à un ordinateur de stocker, récupérer, ajouter, supprimer et modifier des données.

Un SGBD définit ce qu'on appelle le schéma de données ou la structure dans laquelle les données sont stockées.

Parmi les logiciels les plus connus il est possible de citer : MySQL, PostgreSQL, SQLite, Oracle Database, Microsoft SQL Server, Firebird ou Ingres.

Composants d'un SGBD

Les composants du système de gestion sont les suivants :

  • L'administrateur charger de collecter les informations relatives aux fichiers et les mettre à disposition;
  • Le moteur chargé d'appliquer les opérations aux fichiers en tenant compte des informations de l'administrateur;
  • Le traducteur chargé d'assurer le dialogue entre le système et l'usager en mettant à disposition un langage.

Pourquoi utiliser système de gestion de base de données ?

Le SGBD gère trois choses importantes :

  • Les données
  • Le moteur de base de données : permet d'accéder aux données, de les verrouiller et de les modifier
  • le schéma de base de données : définit la structure logique de la base de données.

Les tâches typiques d'administration de base de données prises en charge par le SGBD comprennent la gestion des changements, la surveillance/réglage des performances et la sauvegarde et la restauration.

Les différents modèles de base de données

  • Modèle relationnel : est obtenu en adoptant la structure de table pour la réalisation des relations avec la transformation des entités et des associations en relations.
  • Modèle hiérarchique : le modéle de données est organisé comme un arbre. Les données sont stockées de maniére hiérarchique et réprésentées en usant d'une relation parent-enfant.
  • Modèle orienté réseau : Le modéle en réseau octroit la premission à chaque enfant d'avoir plusieurs parents. Il est utile dans la mesure ou il comble le besoin de relations plus complexes comme les relations plusieurs à plusieurs.
  • Modèle orienté objet : la base de données est définit comme une collection d'objets ou sont stockés toutes les valeurs des données et leurs opérations.

Ces systèmes peuvent être catégorisés selon leur fonctionnement :

  • Système propriétaire : Oracle Database, Microsoft SQL Server, DB2, MaxDB, 4D, dBase, Informix, Sybase.
  • Système libre : MySQL, PostgreSQL, MariaDB, Firebird, Ingres, HSQLDB, Derby, Apache Derby
  • Orienté objet : ZODB, db4o
  • Embarqué : SQLite, Berkeley DB
  • NoSQL : Cassandra, Redis, MongoDB, SimpleDB, BigTable, CouchDB, Couchbase, HBase, LevelDB, RethinkDB, Memcached.
  • Autre système : Access, OpenOffice.org Base, FileMaker, HyperFileSQL, Paradox, Neo4j, Riak, Voldemort

Avantages

  • Offre l'intégrité et la sécurité des données;
  • Offre une variété de techniques pour stocker et récupérer des données;
  • Sait établit les liaisons de façon automatique entre les données;
  • Les programmes peuvent évoluer indépendamment de l'organisation de la base de données;
  • Le volume de données est fonction de la capacité de la machine physique.

Inconvénients

  • La plupart des systèmes de gestion de bases de données sont souvent des systèmes complexes, de sorte que la formation des utilisateurs à l'utilisation du SGBD est requise;
  • Dans certaines organisations, toutes les données sont intégrées dans une seule base de données qui peut être endommagée en raison d'une panne électrique ou la base de données est corrompue sur le support de stockage;
